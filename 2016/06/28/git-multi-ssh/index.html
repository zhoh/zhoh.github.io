有时候我们需要在同一台机器上同时使用多个GIT账号，为了避免冲突，我们需要配置~/.ssh/config文件，其中，多Git账户分两种情况：

  • 不同域名下的不同账户，比如一个github.com域名下的账户，另一个是gitlab.com域名下的账户；
  • 相同域名下的不同账户，如github.com域名下有两个账户；

不同域名不同账号

假如在gitlab.com上有个user1（email1）账号，在github.com上有个user2（email2）账号；

分别生成密钥

使用ssh-keygen -t -rsa -C 'email1'生成SSH Key：id_rsa_gitlab；
使用ssh-keygen -t -rsa -C 'email2'生成SSH Key：id_rsa_github；
为两个账号分别生成两个不重名且可辨识的密钥文件，并将生成的两个SSH Public Key分别添加到Gitlab和Github。
由于默认SSH只会读取id_rsa，为了让SSH识别新的密钥，还需要加到SSH Agent里面

该命令如果报错：Could not open a connection to your authentication agent. 无法连接到ssh agent，可执行ssh-agent bash命令后再执行ssh-add命令；

分别生成密钥

使用ssh-keygen -t -rsa -C 'email2'生成SSH Key：id_rsa_second；
为两个账号分别生成两个不重名且可辨识的密钥文件，并将生成的两个SSH Public Key分别添加到各自Github后台；
